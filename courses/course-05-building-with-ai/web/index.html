The question has changed

"Can an agent do this?" is answered: yes, in a demo. The real question is: will it still work on Tuesday, on real inputs, when someone feeds it a poisoned page?
Making an AI agent look impressive takes an afternoon. Making one you can trust, on the mess and hostility of real data, is the actual job, and it is the job almost everyone skips. About 95% of enterprise AI pilots showed no measurable return (MIT, 2025) and 88% of proofs-of-concept never reached production (IDC, 2026), not because the models were weak, but because nobody scoped, tested, or supervised them. This course teaches the four habits that put your agent in the small minority that ships: scope well, wire well, test well, supervise well. No hype. No code required. Just how to build one that works and cannot hurt you when it fails.

The tools and standards you should actually bookmark

Hand-verified on 13 July 2026. This field moves weekly, so check the vendor's own page when a number matters. The full annotated list is in resources.md.

  • Model Context Protocol (MCP) · the open standard for connecting agents to tools, now under the Agentic AI Foundation (Linux Foundation)
  • Zapier · 9,000+ app integrations plus AI agents, with a free plan to start
  • n8n · 400+ integrations, visual workflows, free to self-host on your own machine
  • OWASP Top 10 for Agentic Applications (2026) · the agent security checklist: goal hijack, tool misuse, memory poisoning, and more
  • The lethal trifecta (Simon Willison) · private data + untrusted content + external communication = a breach waiting to happen
  • LangChain, State of Agent Engineering · where the eval and monitoring gaps are measured
  • METR · independent measurement of what agents can and cannot reliably do
