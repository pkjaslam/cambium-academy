Build a real AI agent

An "agent" is just an AI you set up for a job, gave some knowledge and a few tools, and saved to reuse. The same five moves build one in any tool. Here is exactly how they look in the three most people use. No code.

90-second walkthrough (shown in Claude). The five moves below are identical in every tool.

In Claude, an agent is a Project. Open claude.ai in another tab and follow along. (Free plan lets you try it; saving a Project needs Claude Pro.)

1
Create · Projects → New project

In the left sidebar click Projects, then New project. Name it.

2
The job · Custom instructions

Write a short job description: who it is, what it does, its rules. This is the whole trick.

3
Knowledge · Files panel

Drop in the documents it should always know (up to 20: PDF, text, CSV, images).

4
Tools · Connectors

Add a connector so it can reach your drive, calendar, or the web. A few clicks, no setup.

5
Run it · Start a chat in the Project

Give it a real task. It already knows its job, reads your files, uses its tools.

Grant Helper · Custom instructions
This is move 2, the job
You are a grant-writing assistant for an ecology lab. Draft honest sections from my notes. Never invent citations or results. Ask before assuming a number.
Open Claude →Best for: document-heavy assistants, research, drafting.

In ChatGPT, an agent is a Custom GPT. Open chatgpt.com. (Building a GPT needs a paid plan and the web app.)

1
Create · Explore GPTs → + Create

Click Explore GPTs in the sidebar, then the + Create button.

2
The job · Instructions

Name it, then write Instructions: how it behaves, what to do, what to avoid.

3
Knowledge · Upload files

Under Knowledge, upload the reference files it should draw from.

4
Tools · Capabilities & Actions

Toggle on Web Browsing, Code Interpreter, image generation, or add Actions to call other services.

5
Run it · Preview, then Save

Test it in the preview panel, then Save. Reuse it any time from your sidebar.

GPT Builder · Configure
This is move 2, the job
Name: Grant Helper. Instructions: You are a grant-writing assistant for an ecology lab. Draft honest sections from notes. Never invent citations. Ask before assuming a number.
Open ChatGPT →Best for: a shareable assistant with a clear personality and tools. (Codex is the same idea for coding agents.)

In Zapier, an agent actually does work across your apps, no chat needed. Open agents.zapier.com. This is the closest to "an assistant that runs on its own."

1
Create · + New agent

In the sidebar click New agent. Start from a template or blank.

2
The job · Instructions (plain English)

Write what should set it off and what to do, in normal language. No code.

3
Knowledge · Data sources

Point it at the data it should reference (a sheet, a doc, a knowledge source).

4
Tools · Connect apps (actions)

Give it actions across 7,000+ apps: send an email, update a sheet, post a message.

5
Run it · Set a trigger, turn it on

Pick what triggers it (a new email, a schedule). Now it runs on its own.

The one rule to carry into any tool

An agent is only as safe as the tools you gave it. Give it the fewest tools for the job, and make anything it cannot undo (sending, paying, deleting, posting) ask you first. You do not need it to be perfect; you need its worst mistake to be impossible or easy to catch.
