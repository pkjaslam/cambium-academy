The Practice Range

No scores, no gates, no pressure. Three drills with the answers built in, so you can practice the moves before the honors exam: grade a golden set, sort an agent's permissions, and repair a broken runbook. Reveal the answer whenever you want.

1Grade the golden set

Six agent outputs against their criteria. Call each PASS or FAIL, then check. A shorter answer that drops a required fact is a FAIL, however nicely it reads.

2Sort the permissions

An agent is being wired. For each requested permission, decide: AUTO (safe to run alone), GATE (allow, but a human approves each use), or DENY (it does not need this at all).

3Repair the broken runbook

Here is a runbook for an agent that is about to ship. Three things are dangerously missing or wrong. Name them, then reveal the answer.

RUNBOOK: invoice-helper What it does: Reads incoming invoices and pays the ones it judges legitimate. Owner: (left blank) Tools granted: read_email, read_invoices, pay_vendor (any amount), delete_email Human gate: none, to save time Kill switch: (left blank) Monitoring: none Rollout: switched on for all vendors, day one
The three worst problems:
1. Pay with no human gate. An unrestricted pay tool, any amount, unattended, is the Step Finance failure. Paying should require human approval on every transfer, or the agent should only flag invoices for a human to pay.
2. "Pays the ones it judges legitimate" is unverifiable and high-stakes. Deciding what to pay is exactly the judgment to keep human. The agent may organize and flag; it may not decide to pay.
3. No owner, no kill switch, no monitoring, and a full day-one rollout. Nobody is accountable, nobody can stop it, nobody would notice a failure, and it went straight to everyone with no dry run or shadow stage. Also, a delete_email tool it does not need is extra blast radius to remove.
A safe version: read invoices, flag mismatches and suspected fakes to a human, and never pay anything on its own. Named owner, a kill switch everyone knows, a golden set with a weekly rerun, and a staged rollout.
