THE AGENT WORKSHOP

A complete 90-minute session you can run for your department, your office, or your lab, with nothing but a projector, this website, and a room of people who keep hearing about agents and have never built one. By minute 90, every table has assembled a supervised agent in the Foundry, watched an injection attack die at a permission gate, and left holding a recipe card they made themselves.

THE RUN SHEET · 90 MINUTES

Minute by minute

Timings assume 10 to 30 people and one facilitator. Everything the room sees runs from this site; nothing to install, nothing to sign up for. Each table needs one laptop.

WHENWHAT HAPPENSFACILITATOR NOTES
0:00The hook. Open the Foundry on the projector. Load Naive mode, run the Injection Drill, Run 1. Say nothing until the red draft appears.Let the room read the exfiltration draft in silence. Then: "Nobody in this room would approve that email. The agent never asked."
0:10What an agent is. Five slots on screen: goal, brain, tools, memory, stop rules. A model answers; an agent acts. The loop: think, call, observe, repeat.Point at the trace from the hook, not at abstract diagrams. The evidence is already on screen.
0:25Build one together. Pick the persona closest to the room and load its recipe. Walk the five slots out loud, then run it live.Ask the room to vote each tool's scope before you set it. Disagreement is the lesson arriving.
0:45Tables build their own. Each table loads a persona card below, adapts the goal to their real work, and runs it. One rule: at least one tool on Ask.Circulate. When a permission card pops, make the table decide together before anyone clicks.
1:10Show and tell. Two tables demo their run on the projector, trace visible. The room finds one thing to tighten in each design.Push for scope cuts, not scope adds. "What could you turn OFF and still finish the job?"
1:25The close. Run 2 of the drill: same attack, scoped mode, the gate holds. Read the take-home rule aloud, hand out the cards.End on the rule below. Do not add a second moral. One is enough.
The take-home rule: automate the bounded, gate the rest. An agent earns autonomy one verified run at a time, and it never earns the right to act where you cannot see it.
THE THREE PERSONA CARDS

Pick the one that matches your room

Each card is a complete starting design: load it in the Foundry with one click, or rebuild it by hand from the table, which is better teaching. Every card follows the same shape: a bounded goal, the fewest tools that can finish it, one gated action, and a stop budget.

FACILITATOR CRIB SHEET

What will go wrong, and what to do

PREDICTABLE FAILURES, CALM ANSWERS

  • The cloud brain is slow or busy. It is a free public worker. Switch the brain slot to the Recorded run for the drill moments; they are deterministic on purpose. The lesson lives in the trace, not in the model.
  • No WebGPU on the room's laptops. The Local brain option grays out and says why. Nothing else breaks; use Aira Cloud or recorded runs.
  • The agent goes off protocol. The loop shows a RETRY, then stops honestly if it happens twice. Read that event to the room; it is module 7 (evals) making a live appearance.
  • Someone says "our IT would never allow this." Correct instinct, wrong target: everything here is read-only or draft-only by construction. That conversation about real systems is exactly what modules 3 and 8 of the course are for.
  • A table wants the agent to send the email for real. Perfect. Ask them who signs it when it is wrong. Watch the scope get changed back to Ask by consensus.
  • Wifi dies. The drill, the recorded runs, calc and notes all work offline once the page is loaded. Remote tools will fail visibly and the trace will say so, which is a better lesson than slides about failure.

Where they graduate to
