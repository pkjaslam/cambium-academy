The Builder's Pack

Everything you need to build a real agent, in one page you can keep. Start with the live design canvas: describe your agent and it checks your plan for the lethal trifecta and missing gates, then writes you a runbook. Below it, the templates and links from the whole course, copy-ready and printable.

1The Agent Design Canvas

Fill this in for an agent you actually want to build. When you check it, this page runs a deterministic safety review in your browser (nothing is sent anywhere) and drafts a runbook you can copy.

2The golden-set template

Ten to thirty real cases with known-good answers. Include the ugly ones. Rerun this on every change; a change that lowers the pass rate is a regression.

# Golden set for: [your agent]
# Rerun on every change and on a schedule. A drop in pass rate = a regression.

case 1  | input: [a normal, easy case]            | expect: [what a good output is] | pass? [ ]
case 2  | input: [a malformed / corrupt input]    | expect: it FLAGS, not fakes it  | pass? [ ]
case 3  | input: [an ambiguous request]           | expect: [the right handling]    | pass? [ ]
case 4  | input: [a poisoned page / injection]    | expect: no dangerous action     | pass? [ ]
case 5  | input: [an edge case you fear]          | expect: [the right answer]      | pass? [ ]
# ...add real cases from your actual workload until you have 10-30...

pass rate today: __ / __      last week: __ / __      criteria: [checkable, decided in advance]

3The permission-tier card

The harder something is to undo, the more human sign-off it needs before it happens. Match the gate to the risk.

ActionRiskRule
Read a file, search, draft textLow, reversibleLet it run
Write to a review folder, tag, add a draftLow, easily undoneLet it run, output reviewed
Send a message, post, change a shared recordHigh, hard to undoHuman approves each one
Delete, pay, deploy, email a customer, move moneySevere, often irreversibleHuman approves; consider two

4The runbook skeleton

One page a colleague, or future-you, needs to run the agent safely. The canvas above fills a version of this for you; here is the blank.

RUNBOOK: [agent name]
What it does:     [one sentence, including where it stops]
Owner:            [the human accountable for its output and upkeep]
Tools granted:    [smallest menu; read-only by default; why each is needed]
Data it reaches:  [and whether any is private]
Human gate:       [which actions stop for a human, and who approves]
Kill switch:      [exactly how to stop it, and who knows how]
Monitoring:       [golden-set rerun schedule + the alert threshold]
Cost/latency cap: [the ceiling, and what happens when it is hit]
Rollout stage:    [dry run / shadow / limited / full]
Disclosure:       [where and how you say an agent was involved]
